This activity demonstrates my ability to use Linux commands to audit, analyze, and modify file and directory permissions. I worked with hidden files, directory access controls, and used ls and chmod to meet authorization requirements.

üíª Checking File Permissions

Command:

ls -la /home/researcher2/projects

This command lists all files, including hidden ones, in long format showing permissions, ownership, and file details.

üîç Sample Output Explained

-rw-rw-r-- 1 researcher2 staff 2048 Jun 14 08:00 project_r.txt

This string can be broken down as:

  • - : Regular file
  • rw- : User has read and write
  • rw- : Group has read and write
  • r-- : Others have read only

üîí Changing File Permissions

Requirement: Remove write permission from 'other' on project_k.txt

chmod o-w project_k.txt

This command removes write permissions for users outside the owner/group.

üìÅ Changing Permissions on Hidden File

Requirement: Only user and group can read .project_x.txt

chmod 440 .project_x.txt

This sets read-only access for user and group. No access for others.

Requirement: Only researcher2 should access drafts directory
